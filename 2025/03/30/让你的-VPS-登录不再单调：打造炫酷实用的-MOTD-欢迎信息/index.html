让你的 VPS 登录不再单调：打造炫酷实用的 MOTD 欢迎信息

前言：告别枯燥的 SSH 登录

刚入手 VPS (Virtual Private Server，虚拟专用服务器) 或者开始接触 Linux 系统？很棒！当你通过 SSH (Secure Shell) 远程连接到你的服务器时，通常会看到一个简单的命令行提示符，或者一些非常基础的系统信息。是不是感觉有点单调，而且每次登录都想快速了解下服务器的当前状态？

别担心！我们可以通过一个简单的 Shell 脚本，自定义你的 “Message of the Day” (MOTD)，让它在每次登录时显示丰富多彩、信息全面的系统概览。这不仅看起来更酷，而且非常实用！

想象一下，一登录就能看到：

  • 你的服务器名字和欢迎语
  • 操作系统版本、内核信息
  • CPU 型号、核心数
  • 内存和 Swap 使用情况
  • 磁盘占用率
  • 本地 IP、公网 IP (IPv4/IPv6)
  • 系统运行时间、负载、当前用户数
  • 系统安装时间、启动时间、当前时间

是不是很方便？下面我们就来一步步实现它！

简单解读脚本

对于新手来说，不需要完全理解每一行代码。我们挑几个重点讲讲：

  1. #!/bin/bash: 这是脚本的“魔法开头”，告诉系统要用 Bash 这个程序来运行这个脚本。
  2. # --- Configuration ---: 这里可以调整一些设置，比如 LABEL_WIDTH 控制左边标签的宽度，CURL_TIMEOUT 控制获取公网 IP 的超时时间。
  3. # --- Color Definitions ---: 定义了很多 C_ 开头的变量，它们是特殊的“转义序列”，用来在终端里显示不同的颜色和样式（比如加粗）。C_RESET 用来恢复默认颜色。
  4. print_info() 函数: 这是一个自定义的小工具（函数），用来按固定格式打印每一行信息（标签 + 冒号 + 值），并自动添加颜色和对齐。
  5. # --- Information Gathering ---: 这部分是核心，它运行各种 Linux 命令来获取信息：
    • hostname: 获取服务器名字。
    • grep ... /etc/os-release: 读取文件获取操作系统名称。
    • uname: 获取内核版本和架构。
    • uptime, who, /proc/loadavg: 获取运行时间、登录用户、系统负载。
    • lscpu, nproc, /proc/cpuinfo: 获取 CPU 信息。
    • free: 获取内存和 Swap 信息。
    • df: 获取磁盘使用情况。
    • ip, hostname -I: 获取本地 IP 和网关。
    • curl https://ifconfig.me: 这个命令会访问外部网站来查询你的公网 IP 地址。这需要你的 VPS 能访问互联网，并且安装了 curl 工具。
    • grep ... /etc/resolv.conf: 读取文件获取 DNS 服务器地址。
    • date, who -b, stat: 获取各种时间信息。
    • || echo "N/A": 你会看到很多命令后面跟着这个，它的意思是：如果前面的命令执行失败或没找到信息，就显示 “N/A” (Not Available)，防止脚本出错或显示空白。
  6. # --- Output Section ---: 这部分使用 echo -e (允许解析颜色代码) 和前面定义的 print_info 函数，把收集到的信息按漂亮的格式打印出来。

简单定制 (可选)

  • 调整颜色？ 找到 # --- Color Definitions --- 部分，尝试修改 C_GREEN, C_YELLOW 等变量后面的颜色代码。你可以在网上搜索 “bash color codes” 找到更多颜色选项。
  • 标签太宽/太窄？ 修改 # --- Configuration --- 部分的 LABEL_WIDTH=18 这个数字。
  • 获取公网 IP 太慢？ 如果你的网络不好，可以增加 CURL_TIMEOUT=5 这个数字，或者如果你不需要公网 IP 信息，可以找到获取 EXTERNAL_IPV4 和 EXTERNAL_IPV6 的几行代码，在行首加上 # 把它们注释掉。
  • 不需要某项信息？ 找到 # --- Output Section --- 中对应的 print_info 行，在行首加上 # 注释掉即可。

遇到问题？

  • 没显示任何东西？
    • 确认你编辑的是正确的启动文件 (.bash_profile, .profile 或 .zprofile)。
    • 确认脚本路径 (~/.my_motd.sh) 和添加到启动文件里的路径完全一致。
    • 确认脚本有执行权限 (ls -l ~/.my_motd.sh 查看权限，应该包含 x)。
  • 公网 IP 显示 “curl not found” 或 “Query Failed”？
    • 你的系统可能没安装 curl。尝试安装它：
      • Debian/Ubuntu: sudo apt update && sudo apt install curl -y
      • CentOS/RHEL: sudo yum update && sudo yum install curl -y
    • 如果显示 “Query Failed”，可能是你的 VPS 网络暂时无法访问 ifconfig.me，或者超时时间太短。
  • 安装时间显示 “N/A” 或不准确？ 这是正常的，Linux 没有记录系统“出生日期”的标准方法，脚本只是尽力猜测。
